Evidence Pipeline

Telemetry → Canonical JSON → Signed Dossier → Public Verify

ما مرجع صدور گواهی نیستیم؛ FeedOS شواهد دیجیتالِ قابل‌راستی‌آزمایی تولید می‌کند. صدور رسمی توسط نهاد ثالث معتبر انجام می‌شود.

1) Edge Runtime

  • MQTT / Modbus / LoRaWAN ingestion
  • Buffering & integrity checks
  • Device identity & clock sync

2) Digital Twin & Evidence Compiler

  • Normalization & gaps handling
  • KPIs (FCR, mortality, emissions proxy)
  • Canonicalization for signing

3) Outcome Dossier & QR

  • Signed JSON (HMAC-SHA256)
  • QR → deep link به Claim
  • Tamper-evident structure

4) Verification API

  • /api/verify?id=sample-001 → خواندن Dossier
  • /api/verify?mode=verify&id=sample-001 → اعتبار امضاء
  • Timing-safe comparison
